# 実装計画: 運動習慣スタンプアプリ

## 目標
ユーザーが運動した日に「スタンプ」を押して記録できる、シンプルで継続を促すウェブアプリを作成する。

## 変更内容

### 既存機能の削除
- Book Commerce App 関連のコンポーネントとページ内容を削除します。

### 新規機能実装

#### 1. データ構造
- **StampRecord**: 日付(`YYYY-MM-DD`形式の文字列)の配列として、ブラウザの `localStorage` に保存します。
  - Key: `exercise-stamps`
  - Value: `["2023-10-01", "2023-10-03"]`

#### 2. UIコンポーネント
- **Calendar**: 月ごとのカレンダーを表示。
  - `date-fns` などのライブラリを使用（または標準APIだけで軽量に実装）。
  - 各日付マスをクリックするとスタンプをToggle。
- **Stamp**: 運動完了を示す視覚的なアイコン（「済」「Good」や足跡マークなど）。

#### 3. ページ構成
- **Header**: アプリ名（例: "FitStamp"）と現在の連勝記録などを表示。
- **Home (`/`)**: 今月のカレンダーを大きく表示。「今日のスタンプを押す」という大きなFAB（Floating Action Button）も配置してアクションしやすくする。

### デザイン方針
- **カラー**: エネルギッシュなオレンジやグリーンを基調とする。
- **インタラクション**: スタンプを押したときに「ポンッ」とアニメーションするような心地よさを重視（CSS animation）。

## 検証計画
- `localStorage` にデータが保存され、リロードしても消えないこと。
- 月をまたいでカレンダーを切り替えられること。
