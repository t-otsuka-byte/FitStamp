# 実装計画: PWA (Progressive Web App) 化

## 目標
現在のウェブアプリを、スマートフォンにインストール可能な **PWA** に変換する。
これにより、ネイティブアプリのようにホーム画面から起動し、全画面表示で利用できるようになる。

## 提案される変更

### 1. マニフェストファイルの作成
Next.js の Metadata API (`manifest.ts`) を使用して、以下の設定を行う。
- **name**: "FitStamp"
- **short_name**: "FitStamp"
- **display**: "standalone" (ブラウザのURLバーを非表示にする)
- **theme_color**: オレンジ (#f97316)
- **background_color**: ホワイト (#ffffff)
- **icons**: 各種サイズの設定

### 2. アイコンリソースの追加
- `public/icon.png` (512x512) などを配置するか、`icon.tsx` で動的に生成する。
- iOS 用の `apple-touch-icon` も設定する。
- 今回は `lucide-react` のアイコンを使って `icon.tsx` で動的生成する（画像ファイル作成の手間を省く）。

### 3. ビューポート設定
- スマートフォンでの拡大縮小を防ぎ、ネイティブアプリらしい操作感にする。
- `maximum-scale=1`, `user-scalable=no`

### 4. サービスワーカー (今回は省略または最小限)
- オフライン動作が必要な場合は `next-pwa` や `serwist` が必要だが、まずは「ホーム画面に追加」してアプリライクに動くことを最優先とする（キャッシュ戦略は複雑になるため）。
- 現在の要件では `manifest` だけで「インストール」は可能。

## 検証計画
### 自動テスト
- `npm run build` が通ること。

### 手動検証
- Chrome DevTools の "Application" タブで Manifest が正しく読み込まれているか確認。
- ローカル環境でスマホ（シミュレータ等のブラウザ）からアクセスし、「ホーム画面に追加」ができるか確認。
- 追加したアプリアイコンから起動し、ブラウザの枠がない全画面モードで表示されるか確認。
